# Onix Studios Marketplace

Static frontend for the Onix Studios UEFN product marketplace.

This version has no backend, database, login system, WordPress, Wix, or build step. It is designed for Cloudflare Pages and checkout links from a provider such as Lemon Squeezy, Polar, Gumroad, or Stripe Payment Links.

## Preview locally

Open `index.html` directly in a browser, or run a small local server from this folder:

```powershell
cd C:\Users\suppl\Documents\Websites\OnixStudiosMarketplace
python -m http.server 8080
```

Then open:

```txt
http://localhost:8080
```

## Edit products

Update products in:

```txt
data/products.js
```

Change:

- `name`
- `category`
- `price`
- `theme`
- `status`
- `checkoutUrl`
- `shortDescription`
- `description`
- `includes`
- `specs`
- `media`

The current products are fake examples. When you create live checkout products, replace the placeholder `https://example.com/checkout/...` links with your real buy links.

## Edit product visuals

The preview visuals are generated by CSS, not image files. Each product has a `theme` value:

```js
theme: "arena"
```

Current themes:

- `arena`
- `economy`
- `quest`
- `lobby`

You can adjust their look in:

```txt
css/style.css
```

Search for:

```txt
product-visual--
```

## Edit gallery

Update the fake preview/gallery cards in:

```txt
data/gallery.js
```

Each item can open a product detail modal by matching `productId` to a product in `products.js`.

## Edit FAQ and policies

Update launch support pages in:

```txt
faq.html
policies.html
```

The custom error page is:

```txt
404.html
```

Cloudflare Pages will use `404.html` automatically for missing pages.

## Add real screenshots or videos later

For now, product pages show simulated picture and video slots. Later you can add real image/video fields to `data/products.js` and update `js/script.js` to render those files.

Keep all real images in:

```txt
images/
```

Use clear names like:

- `logo.png`
- `arena-preview-1.jpg`
- `economy-video-thumb.jpg`

Before launch, use:

```txt
LAUNCH-CHECKLIST.md
```

It includes media size targets, analytics setup notes, and the remaining pre-sale checks.

## Update contact email

Change the email in:

```txt
js/script.js
```

Look for:

```js
email: "onixstudiohelp@gmail.com"
```

## Deploy to Cloudflare Pages

1. Push this folder to GitHub.
2. Create a Cloudflare Pages project from the GitHub repo.
3. Use:
   - Framework preset: None
   - Build command: leave blank
   - Build output directory: `/` or project root
4. Connect the domain later in Cloudflare.

All paths are relative, so the site should deploy cleanly from the project root.
